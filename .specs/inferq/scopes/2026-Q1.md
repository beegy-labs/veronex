## Success Criteria

### Core Inference
- [ ] Submit prompt → API Key 인증 → SSE 토큰 스트리밍 수신
- [ ] 3개 동시 요청 → 큐잉 (GPU는 1개씩 처리)
- [ ] 모델 미로드 → 자동 로드 후 추론
- [ ] 메모리 부족 → LRU 비활성 모델 자동 제거
- [ ] Ollama + Gemini 각각 동일 API로 요청 가능

### Multi-Backend
- [ ] `POST /v1/backends` 로 Ollama 서버 등록 (재배포 없음)
- [ ] `POST /v1/backends` 로 Gemini API 키 등록
- [ ] 헬스체크 실패 시 해당 백엔드 자동 OFFLINE 처리

### API Key
- [ ] `iq_` prefix + base62(UUIDv7) 형식 키 발급
- [ ] `X-API-Key` 헤더 인증 동작
- [ ] 요청수 + 토큰(prompt/completion) 사용량 집계

### Web Dashboard
- [ ] `docker compose up` 후 localhost:3000 접속 가능
- [ ] Overview: 요청수, 토큰, 오류율 확인
- [ ] 백엔드 관리: UI에서 Ollama/Gemini 등록/제거
- [ ] API Key 관리: UI에서 키 발급/취소/사용량 확인

## Constraints

- Single GPU (APU, AMD Ryzen AI 9 395+, 96GB unified memory)
- `OLLAMA_KEEP_ALIVE=-1` (greedy allocation)
- ARQ `max_jobs=1` (serial GPU processing)
- PostgreSQL + ClickHouse: 외부 인스턴스 (cluster에 존재)
- Valkey: 기존 cluster 사용 (runtime-storage namespace)
- Rate limiting: 인프라 구축만, 기본값 0 (무제한)
