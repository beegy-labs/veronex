## Success Criteria

### Core Inference
- [x] Submit prompt → API Key 인증 → SSE 토큰 스트리밍 수신
- [x] 3개 동시 요청 → 큐잉 (Valkey RPUSH/BLPOP, max_jobs=1 serial worker)
- [x] 모델 미로드 → 자동 로드 후 추론 (OllamaModelManager.ensure_loaded)
- [x] 메모리 부족 → LRU 비활성 모델 자동 제거 (max_loaded=1, LRU eviction with keep_alive=0)
- [x] Ollama + Gemini 각각 동일 API로 요청 가능

## Constraints

- Single GPU (APU, AMD Ryzen AI 9 395+, 96GB unified memory)
- `OLLAMA_KEEP_ALIVE=-1` (greedy allocation)
- ARQ `max_jobs=1` (serial GPU processing)
- PostgreSQL + ClickHouse: 외부 인스턴스 (cluster에 존재)
- Valkey: 기존 cluster 사용 (runtime-storage namespace)
- Rate limiting: 인프라 구축만, 기본값 0 (무제한)
